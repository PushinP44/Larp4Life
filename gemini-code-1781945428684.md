# HACKATHON PROJECT PROPOSAL: ECOSYSTEM X

* **Competition Track:** Track 1: Little Red Flower Games (Social Impact)
* **Focus Direction:** Direction 2: Biodiversity and Conservation
* **Target Platforms:** PC / Web (HTML5 Browser Deployment)
* **Development Cycle:** 1-Month Production Window
* **Core Visual Architecture:** 2D Top-Down Stylized Pixel Art

---

## 1. Executive Summary & Vision Statement

Ecosystem X: The Last Balance is an AI-driven ecological investigation and restoration game that transforms complex biodiversity dynamics into an immersive, deductive puzzle adventure. Designed to run natively in standard web browsers, the project places players in the role of an Environmental Field Agent deployed to localized biomes on the brink of total collapse. 

Rather than relying on passive text lectures, the game uses procedural generation frameworks to construct completely unique biological networks for every playthrough. Players must actively explore the environment, map out hidden dependencies between species, isolate environmental stressors, and deploy precise interventions to restore equilibrium before the systemic health threshold drops to zero. 

The defining vision of Ecosystem X is to demonstrate that nature is not a collection of isolated organisms, but a dynamic, interconnected network where every single species serves as a critical structural node.

---

## 2. Problem Statement & Social Impact Alignment

Global biodiversity loss is accelerating at an unprecedented rate, yet public engagement remains hindered by the "invisible" nature of systemic ecological collapse. While the average individual understands isolated issues like pollution or endangered species, very few grasp how a minor disruption in a primary trophic layer can trigger a catastrophic cascade throughout an entire regional food web. 

Ecosystem X directly addresses this awareness gap by translating the abstract principles of conservation biology into highly visible, real-time gameplay feedback loops. By navigating simulated ecological crises, players develop systemic thinking patterns, building an intuitive understanding of:

* Trophic cascades and food web vulnerabilities.
* The critical role of specialized pollinators and keystone species.
* The compounding, non-linear long-term consequences of human industrial intervention.
* The balance required to deploy sustainable restoration solutions without creating secondary disruptions.

## 3. Core Gameplay Loop & Mechanical Architecture

The foundational gameplay is driven by a continuous cycle of scientific exploration, data analysis, and environmental management:

Explore Regions ➔ Scan Wildlife ➔ Map Adjacency Graph ➔ Isolate Stressors ➔ Deploy Interventions ➔ Evaluate Results

### Exploration and Sampling Layers
Players navigate a continuous 2D top-down grid map representing a specific threatened biome (e.g., Coastal Wetlands or Mountain Forests). Visual indicators, such as desaturated tile variations, decaying foliage sprites, or reduced wildlife movement, guide the player toward localized zones of high ecological stress.

### The Ecological Investigation Interface
Equipped with a scanner tool, the player collects biometric data from active plant and animal entities. Each successful scan populates a dynamic, in-game Field Notebook. Initially, the relationships between species are hidden. As the player observes interactions over time (e.g., an herbivore consuming a specific seed node), the engine draws explicit connection lines within the player's interface, revealing the hidden food web.

### Targeted Restoration Mechanics
Once a root cause of decline is identified, players spend collected environmental resources to execute localized actions:
* Bioremediation: Deploying targeted filtration methods to reduce regional toxicity levels.
* Population Rebalancing: Carefully reintroducing native species or managing invasive species nodes.
* Habitat Stabilization: Declaring protected zones to increase the baseline carrying capacity of specific tiles.

## 4. Technical Architecture: The Procedural Ecological Network

The core backend engine relies on a deterministic Directed Acyclic Graph (DAG) generated at the initialization of each game world:

[Environmental Factor: Pollution (L)] ➔ [Primary Producer: Orchid] ➔ [Primary Consumer: Bee] ➔ [Apex Predator: Hawk]

### Mathematical State Tracking
Every biological node i within the generated graph tracks its population state P_i across discrete time steps t using localized variables for reproduction capacity r_i, natural mortality m_i, and an environmental stress modifier S_i(L), where L represents the regional pollution metric. The engine processes these variables through a clean, predictable calculation loop:

P_i(t+1) = max(0, P_i(t) + r_i * P_i(t) * (1 - S_i(L)) - Sum[beta_ji * P_j(t) for j in Predators])

### Market Hysteria and Panic Loops
To eliminate the need for unstable, runtime text processing (NLP), social consequences are handled through a systematic Market Hysteria State Machine. If a primary habitat node collapses, a regional panic value increases. 

This causes local village NPC shopkeepers to change their state: trade costs for restoration tools increase significantly, base item availability drops, and dialogue branches automatically swap to pre-rendered, panicked data strings generated by the AI development tools. Order is restored only when the player brings the local ecosystem back to a stable milestone tier.

## 5. End-to-End AI Integration Strategy

Ecosystem X maximizes the capabilities of artificial intelligence by embedding it deeply into the production and asset delivery pipelines, ensuring a highly polished final runtime.

### Development-Time Content & Code Generation (Tencent Cloud CodeBuddy)
* Algorithmic Co-Pilot: CodeBuddy is used to architect the complete graph data structures, grid pathfinding mechanics, and state-checking validation routines. The complete conversation log is preserved as a core artifact for the hackathon judging panel.
* Data Matrix Expansion: AI agents generate the massive, offline JSON database containing distinct species profiles, localized behavioral traits, and state-triggered dialogue trees for the village NPCs, removing the need for live runtime API calls.

### Visual Asset Pipeline (Miora)
* Consistent Pixel Art Style: Miora is used to generate the complete environmental asset pack, including modular top-down terrain tile sets for multiple biomes, UI elements for the Field Notebook interface, and animated sprite frames for generated wildlife nodes.

### Audio Pipeline Automation (VoxFlow Studio)
* Dynamic Audio Scaling: VoxFlow Studio creates the audio assets for the project. The game engine dynamically layers these tracks at runtime: as an ecosystem transitions from Pristine to Toxic, the background audio seamlessly crossfades from bright melodies to sparse, distorted ambient soundscapes.

## 6. 30-Day Minimum Viable Product (MVP) Roadmap

### Phase 1: Core Architecture & Graph Engine (Days 1–7)
Establish the unified global state storage systems and perfect the graph parsing logic. CodeBuddy generates the initial scripts to verify that generated food chains are mathematically stable before rendering.

### Phase 2: Grid Navigation & Discovery Systems (Days 8–15)
Build the top-down 2D canvas rendering loops. Implement player movement, tile interaction, and scanner mechanics that feed directly into the Field Notebook UI.

### Phase 3: Intervention UI & Hysteria Loops (Days 16–22)
Integrate the restoration tool systems and link habitat status directly to the NPC shop pricing models. Implement the pre-baked panic dialogue decks to simulate community reactions.

### Phase 4: Polish, Optimization, and Deployment (Days 23–30)
Import all AI-generated art assets from Miora and soundscapes from VoxFlow Studio. Run end-to-end playtesting to ensure flawless compatibility across standard web browsers.
