# Project Skill: AI Field Report (End-of-Run Recap)

**Trigger:** Building the win/lose cards' debrief (Phase 3, Prompt P10) and `data/report-fragments.json`.
Covers the **Worldbuilding & Story** AI module (40 pts) AND **Theme Alignment** (30 pts) at once.

> **Why this exists / why it scores.** Judges reward AI that feels *intelligent and original*, not just
> decorative. A recap that names the player's *actual* mistakes ("your Painted Stork collapsed on Day 14
> because you treated the symptom — culling shrimp — instead of the runoff feeding the seagrass crash")
> reads as a smart, responsive AI narrator. We get that effect **without breaking the offline +
> determinism laws (Rule 01/02): no runtime LLM.** The report is composed at runtime from (a) the
> player's terminal game state — already fully in `GameState`, no new schema needed — and (b)
> AI-pre-baked sentence fragments generated at dev time by CodeBuddy Genie. Same state ⇒ same report.

## What the report says (derive 100% from terminal state — no event log needed)
Read these straight off `GameState` at win/lose:
- Outcome: `flags.win` / `flags.lose`; `meta.day_count`, `meta.collapse_timer`, final `meta.ecosystem_health`.
- Which **keystones** survived vs went `extinct` (`world.nodes[*].keystone && status==='extinct'`).
- The **root stressor**: the `stressor` node whose tile `L` was highest at end (or lowest if cleaned).
- **Player approach**: did they ever `applyBioremediation` (L dropped) vs only `applyRebalancing`?
  Infer from final tile `L` + `notebook.revealed_edges` coverage (% of edges discovered).
- **Investigation depth**: `notebook.discovered_nodes.length / total nodes`.

## Skill Prompt — pre-baked fragments (run in CodeBuddy Genie / WorkBuddy)
```
Write reusable recap sentence fragments for an ecological-restoration game's end-of-run "Field
Report." Output JSON keyed by situation; each key holds 3-4 interchangeable one-sentence variants
(field-biologist logbook tone, factual, never preachy). I will select one deterministically and
splice in the player's real numbers (day count, species names, health).

Keys required:
  "win_fast"            (restored well within the timer)
  "win_close"           (restored with <=3 days to spare)
  "lose_timeout"        (timer hit 0, biome never reached Pristine)
  "lose_keystone"       (a keystone went extinct -> cascade)
  "lesson_root_cause"   (praises fixing the stressor at the source)
  "lesson_symptom_only" (player only rebalanced populations, never bioremediated -> warns this is
                         treating symptoms; the real-world parallel)
  "lesson_shallow_scan" (player discovered <50% of the dependency web before acting)
  "lesson_deep_scan"    (player mapped most of the web first -> good systemic thinking)
Each fragment may use these placeholders: {days} {keystone} {stressor} {health} {edgePct}.
Output one JSON object: { "<key>": ["variant 1", ...], ... }. Save to data/report-fragments.json.
```

## Skill Prompt — compose it (run in CodeBuddy)
```
Add buildFieldReport(state) to a new module recap.js for Ecosystem X (offline, deterministic).
- Load data/report-fragments.json via ai_content.js (try/catch + small inline fallback).
- Inspect terminal state (outcome, day_count vs collapse_timer, extinct keystones, highest-L
  stressor node, % edges revealed, % nodes discovered, whether any bioremediation happened).
- Pick ONE outcome key + ONE lesson key using a deterministic index seeded by meta.seed
  (NO Math.random), then substitute {days}/{keystone}/{stressor}/{health}/{edgePct}.
- Return { title, outcomeLine, lessonLine, stats:{...} }. Render it inside the existing win/lose
  card (Architecture Law 1: still an overlay card, not a new screen) in the serif codex font,
  with a "Generated by CodeBuddy Genie" attribution badge.
console.assert: same seed + same playthrough -> identical report string.
```

## Rules
- **No new gameplay screen** — the report renders inside the existing win/lose overlay card (Law 1).
- **No runtime LLM** — fragments are pre-baked AI JSON; selection + splicing is deterministic (Law 2).
- Derive everything from terminal `GameState`; do **not** add a per-action event log unless a future
  feature needs one (keeps the schema and the save file lean).
- Keep the AI attribution badge — it is the visible evidence judges look for on the 40-pt dimension.

## Flags
- Any `fetch` to a live model at runtime → violates Rule 01. Pre-bake instead.
- A report that's identical regardless of how the player played → defeats the purpose; it must name
  the real root stressor and the real keystone outcome.
- Selection via `Math.random()` → determinism break; index off `meta.seed`/`meta.day_count`.
