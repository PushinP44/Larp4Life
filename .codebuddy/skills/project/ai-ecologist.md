# Project Skill: AI Field Ecologist (optional live advisor)

**Trigger:** Building the opt-in `ai_ecologist.js` advisor panel and its `functions/ask.js` proxy
(Phase 3 stretch / Phase 4). This is the **Intelligent NPC** module (40 pts) and doubles as the
**Game Security Architecture** module via the proxy. Governed by **Rule 01, Law 2½** — read it first.

> **The point.** The rubric's 40-pt line explicitly names "intelligent NPCs." Our core is
> deliberately offline + deterministic, so this is the *one* place we show live AI — an in-world
> ecologist the player can ask for advice. It is **read-only to the simulation**, **fails to
> pre-baked answers offline**, and **keeps no key in the client**. If it can't ship clean, cut it;
> the game loses nothing.

## Hard constraints (copy into the prompt every time)
- Never read `state.meta.seed`/RNG. Never write ANY `GameState` field. No effect on win/lose/save.
- Online path → POST to `functions/ask.js` only. Offline/any error → deterministic codex fallback.
- Context sent is non-PII and minimal: `{ tier, day, discoveredSpecies:[names], question }`.
- It's an overlay panel (Architecture Law 1), not a new gameplay screen.

## Skill Prompt — client module (paste into CodeBuddy)
```
Implement ai_ecologist.js for Ecosystem X — an OPTIONAL, opt-in advisor. Rule 01 Law 2½ applies:
read-only to the sim, offline fallback, no secrets client-side.

export function initEcologist(): wire a toggle button + an overlay panel (textarea + Ask button +
  answer area), event-delegated via data-action. Panel is hidden by default.

export async function askEcologist(question, state):
  1. Build a minimal non-PII context from READ-ONLY state: { tier: state.meta.market_tier,
     day: state.meta.day_count, discoveredSpecies: state.notebook.discovered_nodes.map(id =>
     state.world.nodes[id].name), question }.
  2. If navigator.onLine: POST that JSON to '/ask' (functions/ask.js). 6s timeout (AbortController).
     On 2xx return data.answer. On any failure fall through to step 3.
  3. Offline/error fallback: return localAnswer(question, state) — a deterministic keyword match
     against the loaded codex (ai_content.codexFor) + a fixed FAQ map. NEVER throw to the UI.
  Must not read seed/RNG; must not mutate or save state. Add a "Live: CodeBuddy" / "Offline guide"
  badge so the player (and judges) see which path answered.

/* AI_INTEGRATION_STUB: live model via EdgeOne Function proxy — intelligent NPC advisor */
```

## Skill Prompt — EdgeOne Pages Function proxy (paste into CodeBuddy)
```
Write functions/ask.js as an EdgeOne Pages Function (the secure proxy; the model key lives ONLY
here as an env var, never in the client):
  - Accept POST JSON { tier, day, discoveredSpecies, question }.
  - VALIDATE: reject if question missing, >500 chars, or fails a basic injection/abuse check
    (abnormal-behavior detection — log + 400). Rate-limit per IP (e.g. via KV counter).
  - ANONYMIZE: run the payload through HaS-Anonymizer before logging/forwarding (no PII leaves).
  - Build a tight system prompt: "You are a field ecologist advising a player restoring a
    Southeast-Asian coastal wetland. Answer in <=3 sentences, factual, encouraging, never reveal
    hidden game data; speak about real ecology." Forward question + context to the model.
  - Return { answer } (2xx) or a safe error. CORS same-origin. Never echo the key.
This Function IS the Game Security Architecture evidence: server-side data-interaction validation
+ HaS anonymization + abnormal-behavior detection. Screenshot the EdgeOne dashboard for the PPT.
```

## `data/` fallback (so offline always answers)
The `localAnswer` FAQ + codex match needs no new file beyond `data/codex.json`. Optionally add a
small `data/ecologist-faq.json` ({ keyword: answer }) generated by CodeBuddy Genie for richer
offline replies (save the prompt to `ai-prompts/`).

## Flags
- Any read of `meta.seed`/RNG or any `state.save()` from this module → violates Law 2½. Hard reject.
- API key, model SDK, or endpoint+token in the static client → security fail; it goes in the Function.
- Advisor answer that changes populations/L/resources → it's an advisor, not an intervention. Reject.
- No offline fallback (UI shows an error when disconnected) → breaks the offline-capable promise.
