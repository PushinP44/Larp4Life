# Project Skill: AI Content Generation (Codex + Dialogue Pools)

**Trigger:** Generating the offline AI JSON (`data/codex.json`, `data/dialogue.json`) and wiring `ai_content.js` (Phase 3). Covers the **Worldbuilding & Story** AI module (rubric, 40 pts).

> All AI text is generated AT DEVELOPMENT TIME with CodeBuddy Genie / WorkBuddy and baked into JSON. The game NEVER calls an LLM at runtime (offline + determinism laws).

## Prompt A — species codex (run in CodeBuddy Genie / WorkBuddy)
```
Generate educational codex entries for an ecological game set in a Southeast Asian coastal
wetland. For EACH species below, write 2–3 sentences that:
1. Name the real species + scientific name.
2. Describe one real biological adaptation.
3. State exactly what human activity (runoff, pollution, habitat loss) threatens it, with a
   concrete real data point where possible.
Tone: factual field-biologist logbook, slightly melancholic, never preachy. ≤180 words each.

Species (match the biome template node ids):
- n_seagrass  — Seagrass (Halophila / Enhalus); keystone primary producer
- n_shrimp    — Mangrove/banana shrimp (Penaeus); key consumer
- n_heron     — Painted Stork (Mycteria leucocephala); apex predator, keystone

Output JSON array: [{ "nodeId": "n_seagrass", "name": "...", "lore": "..." }, ...]
End each lore with one line tying in-game status to the real world.
```

## Prompt B — Market Hysteria dialogue pools (run in CodeBuddy Genie / WorkBuddy)
```
Write villager-vendor dialogue for an ecological restoration game with 4 ecosystem-health
tiers. Give 5 short lines per tier (1 sentence each), reflecting the community's economic mood
as the wetland's health changes. No preaching; show panic→relief through everyday voice.
Output JSON: { "Toxic":[...5], "Degraded":[...5], "Recovering":[...5], "Pristine":[...5] }
```

## Prompt C — wire ai_content.js (run in CodeBuddy)
```
Implement ai_content.js for Ecosystem X (offline, no network):
- export async function loadContent(state): fetch('data/codex.json') and 'data/dialogue.json'
  with try/catch; on failure use a small hardcoded fallback so the game still runs.
- export function codexFor(nodeId, state): returns the lore string (or a discovery hint if
  not yet discovered).
- export function initArtLayers(): preloads tile/sprite images from /assets; if an image 404s,
  the renderer's CSS-color fallback stays active (game must never break on a missing asset).
  /* AI_INTEGRATION_STUB: Miora/MPS — generated tileset + sprites */
- export function initAudio() + setHealthAudio(H): crossfades pre-rendered tier tracks using
  Web Audio gain nodes as H changes (no abrupt cuts).
  /* AI_INTEGRATION_STUB: VoxFlow — tier ambient layers + victory theme */
Cache codex/dialogue in memory after first load. Deterministic selection (no Math.random).
```

## Display + attribution rules
- Show codex in a serif font (`font-family: Georgia, serif`) to distinguish lore from UI.
- Put a small "Generated by CodeBuddy Genie" / "Art: Miora" / "Audio: VoxFlow" attribution in the credits and notebook — judges look for AI-module evidence.
- Keep the generation prompts above in the repo (e.g. `/ai-prompts/`) as proof of AI authorship for the conversation-history submission.
